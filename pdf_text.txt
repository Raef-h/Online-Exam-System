Step 1 – Creating a text file for the exam questions.  
In order to create a new exam,  the exam questions and answers must be imported  from a text file . 
Therefore, a text file must be created for the exam.  Each line  in the text file represents a  single 
question in the exam. The content of the line depends on  the type of question as follows:  
• Question Type:  which can be either TF for True/False questions, or MCQ for multiple 
choice questions.  
• A required  semicolon.  
• Question Text : the actual text of the question . 
• A required  semicolon.  
• In case the question is of type TF: 
o The correct answer  must be provided next, which can be either TRUE or FALSE. 
• In case the question is of type MCQ, four choices must be given in the line as follows : 
o The text of the first choice  is given + a required semicolon  + either CORRECT  or 
INCORRECT  depending on whether this is the correct choice + a required 
semicolon . 
o The text of the second choice is given + a required semicolon  + either CORRECT  
or INCORRECT  depending on whether this is the correct choice + a required 
semicolon . 
o The text of the third choice is given + a required semicolon  + either CORRECT  or 
INCORRECT  depending on whether this is the correct choice + a required 
semicolon . 
o The text of the fourth choice is given + a required semicolon  + either CORRECT  or 
INCORRECT  depending on whether this is the correct choic e.

Step 2.B – Initiating the server for online exams . 
In addition to step 2.A, when  the user clicks “Start Exam Server ”, the application must create a new 
thread responsible for handling messages from students  using Java Sockets ( TCP/IP). This thread 
must in a loop: 
• Listen for incoming client connections  (i.e., students)  using port 3000 .  
• Spawn a new thread for handling the new connection.   
The spawned thread for each connection must use sockets as follows :

 • The server receives as a string the student’s ID. The server must store this information in an 
appropriate collection. The thread then sends back to the client a list of the available exams 
(i.e., course names).  The server waits for the next message from the client.  
• The client picks one of the exams (see Client-Side details below) and sends a message to 
the server containing the desired exam.  
• The server enters a loop, in which it sends the exam questions  of the selected exam , one by 
one, to the client as follows:  
o The server sends the next exam question to the client. Note: the server should not 
send any answers to the client . In addition, th e server should send only one object 
to the client  containing the TF/MCQ details. Sending multiple objects  in different 
messages  (e.g., choices of MCQ in different messages ) is not accepted.  
o The client sends the answer to this question to the server . The server updates the 
score for this exam for this client /student (1 point for each question) . 
o The server repeats the previous two steps until the exam is completed. There is no 
need to allow the client (student) to go back ward to previous questions.  
• The server sends a message to the client with the final score.  The server also inserts into the 
database the score of this student.  
Note: to enable this communication, you are allowed to send objects of any type that you choose 
(even create new types). Ensure that no answers are accidentally sent to clients.  You can maintain 
any nessary collections to store and track exam sessions (e.g. students taking exams).  
The server should not allow any client/student after the exam is finished (60 minutes after creation). 
If a student’s connection is interrupted  during the exam, the student should be allowed to resume 
the exam from the last question before the interruption, given that the exam has not finished.

Database  
===============================================================================  
In this project, you should use a MySQL database to store exam results. The easiest way to install 
MySQL on your device is by using XAMPP ( https://www.apachefriends.org/ ). The database is 
installed at the server side/node.  
After a student completes an exam, the server must store the results in a table  named 
ExamResults. The table consists of the following  columns : 
• ID (int): an autoincrement primary key.  
• Student ID ( text): the ID of the student . 
• Course Name (text): the name of the course  
• Year (text): the exam year  
• Semester (text): the semester.  
• Score (int ): the score the student received.
